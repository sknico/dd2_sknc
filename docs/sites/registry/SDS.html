Population

The DD2 data are uploaded to servers at The Danish Health Data Authority (SDS), and the validity of the CPR numbers is checked before encryption. The dataset DS_EXT_DD2_POP includes all individuals with a valid CPR number. Individuals with a non-valid CPR number are still included in the uploaded data, e.g. in DS_EXT_DD2CORE, but they will have a missing encrypted CPR number (CPR_ENCRYPTED) and can thus not be linked across data sources. When DCE had local registry data, DD2 would be notified about non-valid CPR numbers, but this is no longer possible as it is not allowed to identify the individuals based on individual-level data on the servers.

The uploaded data files (egne data) have the prefix DS_EXT_. Data are available in 2 versions; the initial upload version (without suffix) and the updated versions (with suffix _230601, _231026, etc., corresponding to the date of the update on SDS). The registry data (views) will always be based on the newest population, i.e., individuals in the newest version of DS_EXT_DD2_POP.

Rawdata available on SDS

Data on SDS are based on data views, i.e., DCE have access to data updated (almost) on a day-to-day basis. The update file (SAS metadata.OpdateringsOversigt) contains the time of the current update for each dataset in the library. For research projects, in order to be able to reproduce results, remember to save a copy of the research data in a suitable project folder on SDS.

Note

Using SAS, if you extract data from views you will likely save a lot of time by writing code that is proper sql, rather than using SAS-specific syntax. Please use syntax as in Example 1, rather than Example 2 or Example 3.

Example:
1)
proc sql;
  create table x as
  select *
  from data.lab_lab_dm_forsker
  where analysiscode like [npucode1%] or ... or analysiscode like [npucodeN%];
quit; * takes ~10-20 seconds to run
  
2)  
proc sql;
  create table x as
  select *
  from data.lab_lab_dm_forsker(where = (analysiscode in: ([list of npucodes])));
quit; * takes ~40 minutes to run

3)
proc sql;
  create table x as
  select *
  from data.lab_lab_dm_forsker
  where analysiscode eqt [npucode1] or ... or analysiscode eqt [npucodeN];
quit; * takes ~40 minutes to run

Further documentation on the main data sources can be found here:

Access to data

All DD2-researchers with access to FSEID-00006014 can (in principle) work with all the datasets. As these datasets are considered rawdata, they cannot be edited/changed/moved/renamed/etc. To access data, please do the following:

  • SAS: The data are already available in the libraries in06014 and metadata

  • STATA: Run the (relevant parts of the) DO file located at F:\Projekter\FSEID00006014\DB Connections FSED00006014.do

  • R: Run the (relevant parts of the) R file located at F:\Projekter\FSEID00006014\DB Connections FSED00006014.R

In each research project, it is the individual researchers’ responsibility to ensure data usage is allowed.
