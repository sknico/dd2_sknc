Danmarks Statistik (DST)/Statistics Denmark

Statistics Denmark (DST) Projektnr. 708637 and transfer agreement FSEID-00006159 at The Danish Health Data Authority (SDS).
Locally at Department of Clinical Epidemiology (DCE), data are stored here:
O:\HE_KEA-DATA-RAW0050\Forskermaskiner\DST - projektnr 708637 (SDS FSEID-00006159)
“Udtræksbeskrivelser” can be downloaded from Danmarks Datavindue (DDV).

The project is placed under the “Projektdatabaseordning (705004)”. The DD2 projects itself has the project number 708637 (via 705004), and the agreement with DST makes registry data available. Data from Det Psykiatriske Centrale Forskningsregister, Laboratoriedatabasens Forskertabel, Overvågningsdata - COVID-19-testsvar fra SSI, Det Danske Vaccinationsregister - COVID-19-vaccinationsdata fra SSI, and Cancerregisteret are tranferred from SDS (FSEID-00006159). Data from Lægemiddelstatistikregisteret are included via DDV, but based on the agreement with SDS (FSEID-00006159).

Population

The DD2 data are uploaded to DST, and the population include all individuals in DD2CORE (encrypted CPR). Linked registry data and based on the individuals in the dataset pniveau. Registry data are available in the DST folders (and subfolders):
E:\ProjektDB\KEA\Rawdata\708637\Grunddata
E:\ProjektDB\KEA\Rawdata\708637\Eksterne data

Formats

On DST formats from DST and International Diabetic Neuropathy Consortium (IDNC) should be included manually. Here is an example of how it can be done in SAS

libname fmt "E:\Formater\SAS formater i DAnmarks Statistik\FORMATKATALOG" access=readonly;
options fmtsearch=(fmt.disced fmt.times_personstatistik fmt.dream fmt.statistikbank);

libname idnc "E:\ProjektDB\KEA\Workdata\708637\Common\IDNC formats" access=readonly;
options fmtsearch=(idnc.idnc_formats);

Data uploads

The first data were uploaded in 2022 when the server access was initially established. Since then, DST has changed their system to DDV. In November 2024, we had to make a “genindstilling” via DDV and in principle make a new data agreement. We used this opportunity to make some changes in the data structure (e.g., from IL6_TNF to BIOMARK and GENETIK)
